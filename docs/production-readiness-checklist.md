# Coconique 本番前チェックリスト

更新日: 2026-06-17  
対象: βリリース前の最終確認

## 0. 判定方針

- P0: 本番前に必ずOK。未完了ならリリースしない。
- P1: βでは運用回避可能だが、Issue化しておく。
- P2: リリース後改善でよい。

βリリースOK条件:

- P0がすべてOK
- Stripe test/liveの設定ミスがない
- Didit workflow / webhook / return URL が本番URLで動く
- 決済・本人確認・BAN・通報・退会に致命的な抜けがない
- 管理者が危険ユーザーを止められる
